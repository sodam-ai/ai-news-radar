## Phase 1: MVP — 수집 + 요약 + 분류

### 목표
Phase 1이 끝나면: AI 뉴스 소스를 등록하고, 자동으로 수집·요약·분류된 뉴스를 대시보드에서 카테고리별로 볼 수 있다.

### 기능
- [ ] 기본 프리셋 소스 15개 자동 등록 (영어 10개 + 한국어 5개)
- [ ] 뉴스 소스 추가 등록/삭제/관리 (RSS URL 입력, 활성/비활성 전환)
- [ ] 자동 크롤링 스케줄러 (설정된 주기로 새 글 수집)
- [ ] Gemini 배치 처리 (글 5~10개를 1회 API로 요약+분류+중요도+감성+키워드 한번에)
- [ ] AI 요약 + 카테고리 분류 + ⭐중요도 점수 + 😊감성 분석 + 키워드 추출 (배치 처리)
- [ ] 요약 + 자세히 보기 (3줄 요약 기본 → 클릭하면 원문 전체 펼쳐보기)
- [ ] 중복 뉴스 AI 병합 (같은 뉴스 자동 감지 → 대표 1개 + "▶ N개 매체 추가 보도")
- [ ] "오늘의 브리핑" 자동 생성 (매일 TOP 5 AI 뉴스 1페이지 요약)
- [ ] 키워드 워치리스트 (관심 키워드 등록 → 해당 뉴스 하이라이트)
- [ ] 인앱 리더 뷰 (대시보드 안에서 광고 없이 원문 읽기)
- [ ] URL 바로 접속 (제목/링크 클릭 시 새 탭으로 원문 사이트 바로 이동)
- [ ] 타임라인 뷰 (시간순 뉴스 흐름 — 오늘/어제/이번주 그룹핑)
- [ ] 다크모드 (다크/라이트 전환)
- [ ] 내보내기 (브리핑·북마크를 Markdown/PDF로 내보내기)
- [ ] 웹 대시보드 UI (브리핑 + 목록/타임라인 뷰 + 카테고리/감성 필터 + 다크모드 + Streamlit 2026 동적 탭)
- [ ] 실시간 대시보드 자동 갱신 (st.fragment + auto_rerun으로 새 뉴스 수집 시 자동 업데이트)
- [ ] Gemini 스마트 모델 라우팅 (단순 분류→Flash-Lite 1000회/일, 요약/브리핑→Flash 250회/일)
- [ ] Gemini Context Caching (시스템 프롬프트 캐싱 → 토큰 비용 90% 절감)
- [ ] Gemini 멀티모달 분석 (뉴스 속 이미지/차트를 AI가 분석 → 요약에 시각 정보 반영)

### 데이터
- Source, Article (+ image_analysis 필드), Summary, Category

### 인증
- 없음 (개인용)

### "진짜 제품" 체크리스트
- [ ] 실제 RSS 피드에서 데이터 수집 (목업 데이터 X)
- [ ] 실제 Gemini API 연동 (하드코딩된 요약 X)
- [ ] 로컬에서 안정적으로 실행됨
- [ ] 브라우저에서 대시보드 접속 가능 (http://localhost:6601)

## Phase 2: 확장 — 알림 + 검색 + 북마크

### 전제 조건
- Phase 1이 안정적으로 동작하는 상태

### 목표
Phase 2가 끝나면: 중요 뉴스를 실시간으로 알림 받고, 매일 아침 다이제스트 이메일을 받고, 과거 뉴스를 검색하고, 중요 글을 북마크할 수 있다.

### 기능
- [ ] 실시간 알림 (중요 뉴스 감지 시 데스크톱 알림 또는 이메일)
- [ ] 데일리 다이제스트 이메일 발송 (매일 아침 정리된 뉴스)
- [ ] 글 검색 기능 (키워드 + 카테고리)
- [ ] 북마크 + 메모 기능
- [ ] 읽은 글 히스토리 (이미 읽은 글 표시)
- [ ] 알림 설정 UI (알림 유형·카테고리·시간 설정)
- [ ] 뉴스레터 모니터링 (RSS 없는 뉴스레터를 이메일로 받아 자동 수집·파싱)
- [ ] 다시각도 커버리지 (중복 병합된 뉴스 클릭 → 각 매체 보도 시각 비교 "Full Coverage" 뷰)
- [ ] AI 학습 필터링 (읽기/무시 패턴을 AI가 학습 → 중요도·추천 정확도 향상)
- [ ] 노트 앱 연동 (북마크 글을 Notion DB / Obsidian 폴더로 바로 내보내기)
- [ ] 오프라인 읽기 모드 (WiFi 연결 시 글 미리 다운로드 → 오프라인에서도 읽기)
- [ ] AI 뉴스 채팅 (수집된 뉴스에 대해 자연어 대화 — "이번 주 Claude 관련 뉴스 요약해줘")
- [ ] 음성 브리핑 (매일 브리핑을 edge-tts AI 음성으로 생성 — 출근길에 듣기)
- [ ] 뉴스 소스 자동 발견 (AI가 읽기 패턴 기반으로 새 AI 뉴스 소스 추천)
- [ ] 세컨드 브레인 통합 (단순 내보내기 → AI 자동 태깅 + 관련 노트 연결 + 지식 구조화)
- [ ] 텔레그램 봇 (뉴스 요약+브리핑+AI 채팅 — '/today' 명령)
- [ ] 디스코드 봇 (서버 채널에 자동 브리핑 게시 + AI 채팅)
- [ ] 슬랙 봇 (워크스페이스에서 뉴스 요약+브리핑+AI 채팅)
- [ ] ChromaDB 벡터 검색 (Gemini Embedding 2 기반 시맨틱 검색 → AI 채팅 품질 대폭 향상)
- [ ] 나만의 AI 뉴스레터 자동 발행 (주간 뉴스레터 → Resend API로 이메일 발송)
- [ ] 뉴스 감성 온도계 (감성 데이터 시간별 집계 → Plotly 게이지+시계열 차트)
- [ ] GitHub Actions 서버리스 (cron으로 수집+처리 자동화, 로컬 PC 24시간 가동 불필요)
- [ ] Ollama 로컬 LLM (Llama 3.3으로 완전 오프라인 AI 처리 — API 한도 걱정 제로)
- [ ] 하이브리드 LLM 3단계 라우팅 (Ollama→Flash-Lite→Flash 자동 분기)
- [ ] AI 팩트체크 (뉴스 교차 검증 — "3개 매체 확인 ✅" vs "1개 매체만 보도 ⚠️")
- [ ] 뉴스 디베이트 모드 (AI가 찬성/반대 양쪽 시각 + 근거 3개씩 자동 생성)
- [ ] 게이미피케이션 (AI 뉴스 퀴즈 + 데일리 스트릭 + 레벨/배지/리더보드)
- [ ] 주간 AI 인텔리전스 리포트 (매주 "핵심 트렌드 3개 + 예측 + 행동 제안" PDF 자동 생성)
- [ ] 스마트 알림 (AI가 "이 뉴스를 봐야 하는 이유: ..." 개인화 이유와 함께 알림)
- [ ] 경쟁사 모니터링 (특정 AI 기업/제품 추적 → 관련 뉴스 자동 수집+분석)
- [ ] AI 용어 사전 (뉴스 속 AI 전문 용어를 자동 추출 → 초보자용 쉬운 설명)

### 추가 데이터
- Bookmark, AlertSetting, ReadHistory (Article.is_read 활용)
- NewsletterSource, NoteExport, ReadingLog, UserPreference
- ChatSession, ChatMessage, AudioBriefing, SourceRecommendation
- SentimentTimeline, Newsletter, BotConfig
- FactCheck, Debate
- UserGamification, Quiz, WeeklyReport, CompetitorWatch, GlossaryTerm

### 통합 테스트
- Phase 1의 수집·요약·분류가 여전히 정상 동작하는지 확인
- 뉴스레터 파싱이 기존 RSS 수집과 충돌하지 않는지 확인
- AI 채팅이 ChromaDB 벡터 검색으로 정확한 뉴스를 찾는지 확인
- 텔레그램/디스코드/슬랙 봇이 동일한 응답 품질을 제공하는지 확인
- GitHub Actions 수집 결과가 로컬 수집과 동일한지 확인

## Phase 3: 고도화 — 트렌드 + 분석 + 확장

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중

### 목표
AI 업계 흐름을 한눈에 파악할 수 있는 트렌드 분석 + 더 다양한 소스 수집 + 개인화.

### 기능
- [ ] 주간/월간 AI 트렌드 리포트 자동 생성
- [ ] 키워드 기반 트렌드 차트 (시각화)
- [ ] SNS 소스 추가 (X/Twitter, Reddit 등)
- [ ] 뉴스 연결 지도 (뉴스 간 관계를 Plotly/Pyvis 인터랙티브 그래프로 시각화 — Knowledge Graph)
- [ ] 개인화 추천 (Phase 2 AI 학습 데이터 기반 고도화된 개인 추천 알고리즘)
- [ ] 다국어 지원 (영어→한국어 자동 번역)
- [ ] 크롬 확장 프로그램 (웹 브라우징 중 AI 관련 글을 클릭 한 번으로 수집+분류+요약)
- [ ] AI 타임캡슐 ("1개월 전 오늘 무슨 AI 뉴스?" 회고 + AI 예측 vs 실제 결과 비교)
- [ ] AI 예측 엔진 ("다음 주 핫이슈 3개" 예측 + 나중에 맞았는지 자동 검증)
- [ ] AI 생태계 맵 (AI 기업/기술/인물 간 관계를 인터랙티브 네트워크 맵으로 시각화)
- [ ] AI 멀티 에이전트 시스템 (수집→분석→판단→배포 에이전트 자율 협업, MCP 표준)
- [ ] PWA 모바일 앱화 (Streamlit → PWA 변환, 홈화면 아이콘, 푸시 알림)
- [ ] 대시보드 커스터마이징 (위젯 드래그&드롭 자유 배치)
- [ ] AI 팟캐스트 자동 생성 (NotebookLM 스타일 AI 호스트 2명 토론 팟캐스트 매일 자동 생성)
- [ ] 플러그인 시스템 (확장 가능한 플러그인 API — 커뮤니티 소스 어댑터/분석 엔진 개발)
- [ ] 팀 협업 모드 (대시보드 공유, 뉴스에 코멘트/태그/투표, 팀 큐레이션)
- [ ] AI 콘텐츠 자동 생성 (뉴스 기반 블로그/트윗 스레드/LinkedIn 글을 내 톤앤매너로 작성)
- [ ] 자동 소셜 포스팅 (X/LinkedIn/블로그에 플랫폼별 최적화 포스트 자동 게시)
- [ ] FastAPI 하이브리드 (st.App ASGI로 Streamlit + FastAPI 통합 — 외부 API 제공 가능)

### 추가 데이터
- TrendReport, NewsGraph, AgentLog, SocialPost
- Prediction, EcosystemEntity, PodcastEpisode, Plugin, TeamComment, GeneratedContent

### 주의사항
- SNS API 사용 시 비용 발생 가능 (X API 유료화)
- 트렌드 분석은 충분한 데이터 축적 후에 의미 있음 (최소 1~2개월 운영 후)
- 자동 번역은 Gemini API 호출 횟수를 크게 늘릴 수 있음
- FastAPI 통합은 아키텍처 변경이 크므로 Phase 1~2가 안정된 후에 진행
- st.App은 Streamlit 2026 실험적 기능이므로 안정성 확인 필요
- Knowledge Graph는 최소 1~2개월 데이터 축적 후에 의미 있는 연결이 생성됨
- Plotly/Pyvis 그래프는 데이터가 많아지면 렌더링 성능 주의
- 크롬 확장은 별도 프로젝트(Manifest V3 + TypeScript)로 분리 개발
- AI 타임캡슐은 최소 1개월 이상 데이터가 쌓여야 의미 있는 회고 가능
- 멀티 에이전트 시스템은 MCP 표준이 안정화된 후에 도입 권장
- X API는 유료화되어 비용 발생 가능. LinkedIn은 무료 API 제한적
- PWA는 iOS에서 일부 기능 제한 (백그라운드 싱크, 앱스토어 배포 불가)
